## Skill Any Code 需求文档（以当前实现为准）

Skill Any Code 是一个本地 CLI 工具，用于对大型项目进行结构化代码理解与分析，生成文件/目录级 Markdown 报告，并提供按源码绝对路径查询对应报告文件的能力。

---

## 1. 对外形态

- **显示名称**：Skill Any Code
- **CLI 命令**：`skill-any-code`（别名：`sac`）
- **入口**：`src/cli.ts`（编译产物：`dist/cli.js`）

---

## 2. 配置与环境变量

### 2.1 配置初始化（必须）

- 首次使用必须执行：

```bash
skill-any-code init
```

- 配置文件默认路径：
  - macOS/Linux：`~/.config/skill-any-code/config.yaml`
  - Windows：`C:\Users\<username>\.config\skill-any-code\config.yaml`

### 2.2 环境变量覆盖

- 支持 `SKILL_ANY_CODE_` 前缀覆盖配置，例如：

```bash
export SKILL_ANY_CODE_LLM_BASE_URL="http://127.0.0.1:11434/v1"
export SKILL_ANY_CODE_LLM_API_KEY="your-key"
export SKILL_ANY_CODE_LLM_MODEL="your-model"
export SKILL_ANY_CODE_OUTPUT_DIR="./.skill-any-code-result"
```

---

## 3. 解析与结果落盘

### 3.1 触发解析

解析命令为默认动作（无需子命令）：

```bash
skill-any-code --path /path/to/project --mode auto
```

常用参数（节选）：
- `--path <path>`：项目根路径（默认 `process.cwd()`）
- `--mode <mode>`：`full | incremental | auto`（默认 `auto`）
- `--depth <number>`：解析深度（默认 `-1` 不限制）
- `--concurrency <number>`：并发数
- `--output-dir <path>`：结果输出目录（默认 `./.skill-any-code-result`）
- `--skills-providers <list>`：逗号分隔 providers（`opencode/cursor/claude/codex`）
- `--no-skills`：跳过 Skill 生成
- LLM 参数：`--llm-base-url` / `--llm-api-key` / `--llm-model` 等

### 3.2 默认产物与目录结构契约

- **默认结果目录**：被分析项目根目录下 `./.skill-any-code-result`
- **索引文件**：`analysis-index.json`
- **元数据文件**：`.analysis_metadata.json`
- **目录聚合**：`<dir>/index.md`
- **文件结果**：`<dir>/<file>.md`
  - 若源文件为 `index.<ext>`，则结果必须为 `index.<ext>.md`（避免与目录 `index.md` 冲突）。

---

## 4. resolve 查询能力

根据源码**绝对路径**查询对应的分析报告 Markdown 路径：

```bash
skill-any-code resolve "C:/my-project/src/index.ts" --project "C:/my-project"
```

- **命中**：stdout 单行输出 Markdown 报告绝对路径
- **未命中**：stdout 单行输出 `N/A`

---

## 5. 黑名单与项目级过滤

- **三层合并**：全局 blacklist（配置）→ 项目级 `.skill-any-code-ignore` → 项目 `.gitignore`
- **结果目录自忽略**：默认必须忽略 `.skill-any-code-result/`，避免解析自身产物与进入索引。

---

## 6. LLM 连接校验与缓存

- 解析前必须进行 LLM 连接/配置校验，失败则退出并输出明确前缀。
- 默认缓存目录：`~/.cache/skill-any-code/llm`
- 支持清空缓存：`--clear-cache`
