## Skill Any Code 测试文档（以当前实现为准）

本文档用于定义 Skill Any Code（`skill-any-code` / `sac`）的自动化测试范围与关键契约，确保在重构、性能优化与功能演进中保持对外行为稳定。

---

## 1. 测试分层与目标

- **单元测试**：路径/配置/黑名单/索引读写等纯逻辑正确性（不依赖真实 CLI 交互）。
- **集成测试**：从 `dist/cli.js` 触发端到端流程（init → 解析 → 索引 → resolve），验证文件系统产物与约束。
- **系统测试**：完整用户场景、异常路径、缓存上限、LLM 连接校验等跨模块行为。

---

## 2. 关键对外契约（必须稳定）

### 2.1 配置生命周期

- **未初始化即失败**：配置文件不存在时执行解析应立即失败，并提示先执行 `skill-any-code init`。
- **默认配置路径**：
  - macOS/Linux：`~/.config/skill-any-code/config.yaml`
  - Windows：`C:\Users\<username>\.config\skill-any-code\config.yaml`

### 2.2 默认产物路径与命名

- **默认结果目录**：被分析项目根目录下 `./.skill-any-code-result`
- **索引文件**：`analysis-index.json`
- **元数据文件**：`.analysis_metadata.json`
- **命名冲突策略**：源文件为 `index.<ext>` 时，文件结果必须为 `index.<ext>.md`，目录结果为 `index.md`，两者不得互相覆盖。

### 2.3 resolve 查询行为

- **命中**：stdout 输出对应 Markdown 文件的绝对路径（单行）
- **未命中**：stdout 输出 `N/A`（单行）
- **路径归一化**：输入反斜杠、尾部斜杠等都应正确归一化后匹配索引。

### 2.4 三层黑名单合并

- 合并顺序：全局 blacklist（配置）→ 项目级 `.skill-any-code-ignore` → 项目 `.gitignore`
- 结果目录必须被默认忽略：`.skill-any-code-result/` 不得参与解析与进入索引。

### 2.5 LLM 连接校验与缓存

- 解析前必须进行 LLM 连接/配置校验，失败则退出且输出明确前缀。
- 默认缓存目录：`~/.cache/skill-any-code/llm`
- 支持容量上限与清空缓存（`--clear-cache`）。
