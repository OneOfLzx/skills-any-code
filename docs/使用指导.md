# code-analyze-cli 大型项目代码理解分析工具
独立的大型项目代码理解与分析CLI工具，支持多语言代码解析、自动生成摘要、查询项目结构和代码信息，帮助开发者快速理解陌生项目。

---

## 📋 环境要求
| 软件 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | >= 16.0.0 | 运行环境 |
| Git | >= 2.30.0 | Git相关功能支持 |
| 操作系统 | Windows 10+/macOS 12+/Ubuntu 20.04+ | 全平台兼容 |

## 🚀 安装与运行
### 1. 项目初始化
```bash
# 克隆项目后进入目录，安装所有依赖
npm install

# 构建项目（编译TypeScript到dist目录）
npm run build
```

### 2. 安装使用方式
#### 方式一：全局安装（推荐）
```bash
npm install -g .
# 安装成功后即可使用全局命令
code-analyze --version
# 或简写别名
ca --version
```

#### 方式二：直接运行
```bash
# 用npm脚本运行
npm start -- [命令] [参数]

# 或直接调用编译后的文件
node dist/cli.js [命令] [参数]
```

## 🔧 核心命令说明
### 1. 项目解析命令：`analyze`
用于执行项目代码解析，生成分析结果
```bash
code-analyze analyze [选项]
# 选项：
# -p, --path <path>       指定解析的项目根路径，默认当前工作目录
# -m, --mode <mode>       解析模式：full(全量)/incremental(增量)/auto(自动)，默认auto
# -d, --depth <number>    解析深度，默认-1（无限制）
# -C, --concurrency <num> 并行解析并发数，默认CPU核数*2
# --output-dir <path>     自定义结果输出目录
# --no-confirm            跳过所有确认提示
# --force                 强制解析，忽略未提交变更警告
```
**示例**：
```bash
# 解析当前目录项目
code-analyze analyze

# 解析指定路径的项目，使用全量模式，深度限制为3层
code-analyze analyze -p /path/to/your/project -m full -d 3
```

### 2. 结果查询命令：`query`
查询已经解析完成的项目结果
```bash
code-analyze query <query-path> [选项]
# 参数：
# <query-path>            需要查询的文件/目录相对项目根的路径
# 选项：
# -t, --type <type>       查询类型：summary(摘要)/full(完整信息)/diagram(图表)，默认summary
# -p, --project <path>    项目根路径，默认当前工作目录
# --output-dir <path>     自定义结果输出目录
# -o, --output <format>   输出格式：text/json/markdown，默认text
```
**示例**：
```bash
# 查询src目录的摘要信息
code-analyze query src

# 查询src/index.ts文件的完整信息，输出为markdown格式
code-analyze query src/index.ts -t full -o markdown

# 查询src/service目录的结构图表
code-analyze query src/service -t diagram
```

### 3. 配置管理命令：`config`
管理工具的本地配置
```bash
code-analyze config [选项]
# 选项：
# --list                  列出所有配置
# --set <key>=<value>     设置配置项，例如：--set global.log_level=debug
# --get <key>             获取配置项值，例如：--get llm.apiKey
# --reset                 重置所有配置为默认值
```
**示例**：
```bash
# 查看所有配置
code-analyze config --list

# 设置LLM接口的API Key
code-analyze config --set llm.apiKey=your-doubao-api-key

# 查询默认输出格式
code-analyze config --get global.output_format

# 重置所有配置
code-analyze config --reset
```

## ⚙️ 配置说明
### 配置文件路径
默认配置文件位于：
- macOS/Linux: `~/.config/code-analyze/config.yaml`
- Windows: `C:\Users\你的用户名\.config\code-analyze\config.yaml`
也可以通过`-c, --config`参数指定自定义配置文件路径。
> 💡 首次运行工具时会自动生成包含所有默认值的配置文件，无需手动创建。

### 完整配置项
| 配置分组 | 配置项 | 说明 | 默认值 |
|----------|--------|------|--------|
| **global** | log_level | 日志级别：debug/info/warn/error | info |
| | output_format | 默认输出格式：text/json/markdown | text |
| | auto_confirm | 是否自动跳过确认提示 | false |
| | output_dir | 分析结果存储目录 | ~/.cache/code-analyze/results |
| **analyze** | default_mode | 默认解析模式：full/incremental/auto | auto |
| | default_concurrency | 默认并发数 | CPU核数*2 |
| | default_depth | 默认解析深度 | -1（无限制） |
| | exclude_patterns | 解析时忽略的文件/目录模式 | 默认包含node_modules、dist、.git等 |
| **query** | default_type | 默认查询类型：summary/full/diagram | summary |
| **llm** | baseURL | LLM接口地址 | https://ark.cn-beijing.volces.com/api/v3 |
| | apiKey | 豆包大模型API Key | 空（需要用户自行配置） |
| | model | 使用的大模型名称 | doubao-seed-1-6-251015-251015 |

### 环境变量配置
也可以通过环境变量覆盖配置，环境变量前缀为`CODE_ANALYZE_`，例如：
```bash
export CODE_ANALYZE_LLM_APIKEY=your-api-key
export CODE_ANALYZE_OUTPUT_FORMAT=json
```

## 💡 常见使用流程
1. 首次使用先配置LLM的API Key：`code-analyze config --set llm.apiKey=your-api-key`（API Key可在[火山引擎方舟平台](https://console.volcengine.com/ark/)申请）
2. 进入需要分析的项目根目录，执行解析：`code-analyze analyze`
3. 解析完成后查询结果：`code-analyze query src/xxx`
4. 如果需要调整配置，使用`config`命令修改即可
